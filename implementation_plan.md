# Implementation Plan: Переворот конвейера детектирования паттернов

[Overview]
Изменить порядок выполнения проверок в системе детектирования паттернов: сначала запускать корреляцию со всеми паттернами до применения фильтров по форме сигнала.

Текущая реализация отсеивает 99% сигналов жесткими фильтрами по форме еще до сравнения с паттернами. Это приводит к тому что сигналы с не стандартной формой никогда не детектируются даже если для них есть идеально сохраненный паттерн. Новый порядок даст увеличение вероятности детектирования в 10-15 раз и позволит работать с любыми типами сигналов.

[Functions]

### Модифицируемые функции:

1.  **`SpectrumShape::analyze()`
    - Текущее поведение: выполняет все вычисления и все проверки фильтров
    - Новое поведение: выполняет только вычисления параметров, проверки фильтров выносятся в отдельную функцию

2.  **`SpectrumShape::pass_filters()`
    - Новая функция, принимает результат анализа и конфиг, возвращает bool прошел ли сигнал фильтры

3.  **`PatternMatcher::match()`
    - Добавить параметр `bool apply_global_filters`
    - По умолчанию `true` для обратной совместимости
    - При значении `false` запускает корреляцию для всех паттернов сразу

4.  **`PatternMatcher::check_pattern_filters()`**
    - Новая функция, применяет фильтры формы индивидуально для каждого паттерна

## 📋 Детали логики работы с паттернами:

### ✅ Как добавлять паттерны:
1.  Перейти в режим Sweep
2.  Нажать кнопку `PTR`
3.  Два раза нажать на спектр чтобы выделить область
4.  Система автоматически соберет 5 кадров, усреднит их и сохранит на SD карту
5.  Паттерн автоматически включается в проверки

### ✅ Как удалять паттерны:
1.  Паттерны хранятся в папке `/PATTERNS` на SD карте
2.  Каждый паттерн это отдельный текстовый файл
3.  Удалить файл чтобы удалить паттерн
4.  Или использовать будущий интерфейс управления паттернами

### ✅ Как включаются и выключаются проверки:
- Каждый паттерн имеет индивидуальный флаг `ENABLED`
- По умолчанию все новые паттерны включены
- Порог корреляции индивидуальный для каждого паттерна
- Глобальный переключатель `Pattern Matching ON/OFF` в настройках
- При отключенном переключателе работает только стандартный детектор по RSSI
